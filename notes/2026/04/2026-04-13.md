## 핵심 개념

### revert
Solidity 104는 고급 보안, 가스 최적화, 라이브러리 및 디자인 패턴을 다룬다.(실전 코드 스타일)

revert는 조건이 잘못되었을 때 실행을 즉시 중단하고 모든 상태 변경을 되돌린 뒤 에러를 반환하는 것

---

## 코드 

### revert
```solidity
if (msg.sender != owner) revert NotOwner();
```

### flow
```solidity

// SPDX-License-Identifier: MIT
// MIT 라이선스를 따른 다는 표시(컴파일러 경고 방지 + 법적 표시)

pragma solidity ^0.8.20;
// 버전 0.8.20이상에서 실행

contract SecureValut {
    // custom errors (가스 절약)
    
    error NotOwner();
    // onner가 아닌 사ㅏㄻ이 접근했을 때 발생시키는 에러

    error InvalidAmount();
    // 금액이 0이거나 잘못된 값일 때 사용하는 에러

    // state variables
    address public immutable owner; // 컨트랙트를 배포한 사람 주소 (변경 불가)
    uint256 public totalDeposits; // 전체 입금된 ETH 총합

    mapping(address => uint256) public balances;
    // 사용자별 입금 금액 저장 (주소 -> 금액)

    // events (외부 추적용)
    event Deposited(address indexed user, uint256 amount);
    // 누가 얼마 입금했는지 기록, indexed = 특정 주소 검색 빠르게 가능
    event Withdrawn(address indexed owner, uint256 amount);
    // owner가 얼마 출금했는지 기록

    // Constructor (배포 시 1번 실행)
    constructor(){
        onwer = msg.sender;
        // 컨트랜트를 배포한 주소를 owner로 설정

    }
    .....


    
}

```
